swiftUI使用.sheet显示新窗口

Created By: jas chen
Last Edited: Mar 04, 2020 1:30 PM

SwiftUI’s sheets are used to present new view controllers modally over existing ones, like calling present() on a UIViewController in UIKit. To use one, give it something to show (some text, an image, a custom view, etc), add a Boolean that defines whether the detail view should be showing, then attach it to your main view as a modal sheet.

For example, if you had a detail view like this one:

struct DetailView: View {
    var body: some View {
        Text("Detail")
    }
}

Then you could present it like this:

struct ContentView: View {
    @State var showingDetail = false

    var body: some View {
        Button(action: {
            self.showingDetail.toggle()
        }) {
            Text("Show Detail")
        }.sheet(isPresented: $showingDetail) {
            DetailView()
        }
    }
}

Unlike navigation links, sheets don’t require a navigation view to work.

GitPush出现Writing object 卡住

Created By: jas chen
Last Edited: Feb 14, 2020 7:53 PM

原因

这是由于 Git 的缓冲区导致的，由于缓冲区过小，无法完全容纳本次做的更新操作。故一直卡在某个位置无法继续执行任务。

解决方法

修改缓冲区大小

git config --global http.postBuffer 52428800

500MB : 52428800

1GB : 1048576000

swift基础

Created By: jas chen
Last Edited: Mar 04, 2020 1:30 PM
Tags: swift

输出

默认情况下，terminator的值为"\n", 因此可以不用输入terminator参数。

print("Hello World", terminator: " this is the end of the str")
//输出结果：Hello World this is the end of the str

整数

范围

可以通过访问不同类型的min和max来获取对应类型的最小值的最大值。

let min8Value = UInt8.min
//0
let max8Value = UInt8.max
//255
let min16Value = UInt16.min
//0
//类似可以获得 UInt16、UInt32的最大值、最小值

Int

swift 提供了一个特殊的整数类型Int，长度和当前平台原生字节长相同。

  • 在32位平台上，Int和Int32长度相同
  • 在64位平台上，Int和Int64长度相同

在32位平台上，Int可以存储的范围可以达到 -2147483648～2147483647

UInt

是 swift 提供的一个无符号的UInt，长度与当前平台的原生字节长度相同

浮点数

Double 表示64位浮点数。

Float 表示32位浮点数。

Double的精度很高，至少有15数字，而Float只有6位数字。

类型别名

typealias AudioSample = UInt16
var max = AudioSample.min

元组

元组内的值可以为任意类型，并不要求是相同类型。

var HttpError = (404,"Not Found")

print(HttpError.0)
//可以通过.来访问元组中的内容

var HttpError2 = (Code: "200",description: "ok")

print(HttpError2.description)
//命名之后可以通过名称来访问内容

let (justTheCode,_) = HttpError2
//可以通过_来表示不需要的元素

可选类型

循环

//闭区间
for index in 1...3{
    print(index)
}
//开区间
for index in 1..<5{
    print(index)
}
//循环的降序
for index in (1..<5).reversed() {
    print(index)
}

//
//  ContentView.swift

//
//  Created by jas chen on 2020/2/17.

import SwiftUI

struct ContentView: View {
    @FetchRequest(entity: Student.entity(), sortDescriptors: []) var students: FetchedResults<Student>
    @Environment(\.managedObjectContext) var moc
    let firstNames = ["Ginny", "Harry", "Hermione", "Luna", "Ron"]
    let lastNames = ["Granger", "Lovegood", "Potter", "Weasley"]
    var body: some View {
        VStack{
            List{
                ForEach(students, id: \.id){ student in
                    Text(student.name ?? "Unknow")
                }
            }
            Button(action: {
                let chosenFirstName = self.firstNames.randomElement()!
                let chosenLastName = self.lastNames.randomElement()!
                let student = Student(context: self.moc)
                student.id = UUID()
                student.name = "\(chosenFirstName) \(chosenLastName)"
                try? self.moc.save()

            }) {
                Text("Add")
            }

        }
    }
}

疑难排查

访问127.0.0.1:9870出现permission denied


这是由于权限问题导致的，可以通过在core-site.xml中通过配置相关信息解决：

<property>
        <name>hadoop.http.staticuser.user</name>
        <value>hadoop</value>
</property>

value值就是Hadoop中http访问静态用户名字。

除此之外，对于HDFS的权限配置在 hdfs-default.xml 中是默认开启的。

dfs.permissions.enabled=true #是否在HDFS中开启权限检查,默认为true

解决方法有两种，第一种是修改目录权限

hdfs dfs -chmod -R 755 /

即可解决问题。
第二种是直接配置 Hadoop 的权限配置：

<property>
        <name>dfs.permissions.enabled</name>
        <value>false</value>
    </property>

更推荐使用第一种方式。

IDEA中写MR连接HDFS出现权限问题

使用如下方法设置环境变量：

System.setProperty("HADOOP_USER_NAME","jas"); //设置环境变量，更换用户。解决权限问题。

根据你的实际情况更改环境变量的名字。

Mac搭建Hive环境

安装Hive

brew install hive

配置MySQL

创建hive用户并授予权限

mysql -u root -p

mysql>create database metastore;

mysql>grant all on *.* TO 'hive'@'%' with grant option;

mysql>flush privileges;

mysql>exit;

然后授权，使权限生效。

修改hive的配置文件

修改/usr/local/Cellar/hive/3.1.2/libexec/conf/hive-site.xml信息如下：

<?xml version="1.0" encoding="UTF-8" standalone="no"?>              
<?xml-stylesheet type="text/xsl" href="configuration.xsl"?>
<configuration>
<property>
        <name>javax.jdo.option.ConnectionUserName</name>
        <value>root</value>
    </property>
    <property>
        <name>javax.jdo.option.ConnectionPassword</name>
        <value>root</value>
    </property>
    <property>
        <name>javax.jdo.option.ConnectionURL</name>mysql
        <value>jdbc:mysql://localhost:3306/metastore</value>
    </property>
    <property>
        <name>javax.jdo.option.ConnectionDriverName</name>mysql驱动>程序
        <value>com.mysql.jdbc.Driver</value>
    </property>
</configuration>

javax.jdo.option.ConnectionUserName -- 连接mysql的账号
javax.jdo.option.ConnectionPassword -- 连接mysql的秘密
javax.jdo.option.ConnectionURL -- 对应上一步创建的数据库

在HDFS中创建hive所需要的仓库

hdfs fs -mkdir /tmp
hdfs fs -mkdir -p /user/hive/warehouse
hdfs fs -chmod g+w /tmp
hdfs fs -chmod g+w /user/hive/warehouse

如果没有配置环境变量那么请到HADOOP_HOME中执行上述操作。

这里创建的这些目录是hive默认保存数据的地址，还可以通过配置hive-site.xml修改保存的位置。

<property>
    <name>
        hive.metastore.warehouse.dir
    </name>
    <value>
        /user/hive/warehouse
    </value>
    <description>
        location of default database for the warehouse
    </description>
</property>

配置同组用户有执行权限
hdfs dfs -chmod g+w /user/hive/warehouse

初始化mysql中的数据库warehouse

schematool -dbType mysql -initSchema

启动hive的Metastore Server服务进程

hive --service metastore &

登陆hive客户端

hive

注意⚠️

初始化mysql中的数据库warehouse之前，需要先将和mysql版本匹配的mysql-connector 置于 /usr/local/Cellar/hive/3.1.2/libexec/lib中。

all done✅

Mac搭建Hadoop环境

Mac下可以使用brew安装Hadoop，有关HomeBrew的安装配置请参考Tools/。

下载安装Hadoop

brew install hadoop

稍等片刻即可完成Hadoop的安装。
有关hadoop的文件目录可以使用brew list hadoop

配置 SSH

安装 Hadoop 需要远程登陆的功能，如果不想一直输入密码可以通过配置 SSH 秘钥。

  1. 在系统偏好设置中打开远程登录，位置在 System Preferences -> Sharing中。
  2. 方法二：
sh-keygen -t rsa -P '' -f ~/.ssh/id_rsa # 生成公钥和私钥
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys # 将公钥追
加到文件
chmod 0600 ~/.ssh/authorized_keys # 修改相关权限

可以使用ssh localhost方式进行测试。

配置文件的修改

配置环境变量

修改/usr/local/Cellar/hadoop/3.1.2/libexec/etc/hadoop/hadoop-env.sh：
添加

export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.8.0_212.jdk/Contents/Home

JAVA_HOME根据自己的实际情况进行修改。

core-site.xml

Hadoop的核心配置文件，里面记录了HDFS和MapReduce都需要使用的配置
如：NameNode的信息。

配置内容：

<configuration>
<property>
<!--临时文件 -->
        <name>hadoop.tmp.dir</name>
        <value>/usr/local/Cellar/hadoop/hdfs/tmp</value>
        <description>A base for other temporary directories.</description>
    </property>
    <property>
    <!--namenode 的位置，由于是HDFS和MR都需要使用，因此在core中，而不是在hdfs-site.xml 里-->
        <name>fs.default.name</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>

hdfs-site.xml

这里配置了hdfs的信息，修改其为如下：

<configuration>
<!--每个block重复的次数-->
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
</configuration>

由于我使用的是伪分布，因此设置重复次数为1.

maperd-site.xml

这是MapReduce的配置信息。修改其为如下：


<configuration>
<property>
<!--这是JobTracker的地址，因为只有MR使用，所以他出现在mapred-site.xml中 -->
        <name>mapred.job.tracker</name>
        <value>localhost:9010</value>
    </property>
</configuration>

结束语

至此Hadoop的安装配置过程就全部完成了。

附加信息

启动Hadoop 之前需要先格式化NameNode，方法如下

  1. 进入/usr/local/Cellar/hadoop/3.1.2/bin 中

  2. 执行 hadoop namenode -format

  3. 启动Hadoop：

    ./start-all.sh

  4. 使用jps可以查看各节点启动情况。

ARKit介绍

ARKit可以和SceneKit与SpriteKit无缝结合，或者是与Metal 2配合直接控制渲染。

SceneKit 是iOS游戏世界中的三维世界，相应的SpriteKit是创建二维世界使用的。

功能

追踪

追踪是ARKit的核心功能，也就是可以实时追踪设备。

  • 世界追踪 可以提供设备在物理环境中的相对位置
  • 视觉惯性里程计 可以提供设备所在的位置的精确的视图以及设备朝向，它使用了相机图像和设备的运动数据
    ### 场景理解
    场景理解提供了平面检测等功能
    （我们得有一个平面然后才可以从上面放置物品，对吧？）
    除此之外，场景理解还可以进行光线估算，这会让你的物体和物理世界更匹配。
    ### 渲染Rendering
    通过以上步骤我们已经可以把虚拟物体放在现实世界中，那么最后就是与这个新组成的世界交互，这些都是渲染做到的。
    打个比方，当你移动设备的时候，屏幕里的还会保留在原来的位置上，这就是渲染做到的。

ARKit 是离不开SceneKit和SpriteKit的，ARKit借助他们来渲染场景，本身ARKit仅仅做了数据的处理。
